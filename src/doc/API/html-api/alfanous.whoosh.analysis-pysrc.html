  17  """Classes and functions for turning a piece of text into an indexable stream
 
  18  of "tokens" (usually equivalent to words). There are three general types of
 
  19  classes/functions involved in analysis:
 
  20  
 
  21  * Tokenizers are always at the start of the text processing pipeline. They take
 
  22    a string and yield Token objects (actually, the same token object over and
 
  23    over, for performance reasons) corresponding to the tokens (words) in the
 
  24    text.
 
  25        
 
  26    Every tokenizer is a callable that takes a string and returns a generator of
 
  27    tokens.
 
  28        
 
  29  * Filters take the tokens from the tokenizer and perform various
 
  30    transformations on them. For example, the LowercaseFilter converts all tokens
 
  31    to lowercase, which is usually necessary when indexing regular English text.
 
  32        
 
  33    Every filter is a callable that takes a token generator and returns a token
 
  34    generator.
 
  35        
 
  36  * Analyzers are convenience functions/classes that "package up" a tokenizer and
 
  37    zero or more filters into a single unit, so you don't have to construct the
 
  38    tokenizer-filter-filter-etc. pipeline yourself. For example, the
 
  39    StandardAnalyzer combines a RegexTokenizer, LowercaseFilter, and StopFilter.
 
  40      
 
  41    Every analyzer is a callable that takes a string and returns a token
 
  42    generator. (So Tokenizers can be used as Analyzers if you don't need any
 
  43    filtering).
 
  44    
 
  45  You can implement an analyzer as a custom class or function, or compose
 
  46  tokenizers and filters together using the ``|`` character::
 
  47  
 
  48      my_analyzer = RegexTokenizer() | LowercaseFilter() | StopFilter()
 
  49      
 
  50  The first item must be a tokenizer and the rest must be filters (you can't put
 
  51  a filter first or a tokenizer after the first item).
 
  52  """
