Class TopDocs

source code

object --+
         |
        TopDocs

This is like a list that only remembers the top N values that are added to it. This increases efficiency when you only want the top N values, since you don't have to sort most of the values (once the object reaches capacity and the next item to consider has a lower score than the lowest item in the collection, you can just throw it away).

The reason we use this instead of heapq.nlargest is this object keeps track of all docnums that were added, even if they're not in the "top N".
