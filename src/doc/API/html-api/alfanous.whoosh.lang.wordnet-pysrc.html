 92      """Represents the WordNet synonym database, either loaded into memory 
 93      from the wn_s.pl Prolog file, or stored on disk in a Whoosh index. 
 94       
 95      This class allows you to parse the prolog file "wn_s.pl" from the WordNet prolog 
 96      download into an object suitable for looking up synonyms and performing query 
 97      expansion. 
 98   
 99      http://wordnetcode.princeton.edu/3.0/WNprolog-3.0.tar.gz 
100       
101      To load a Thesaurus object from the wn_s.pl file... 
102       
103      >>> t = Thesaurus.from_filename("wn_s.pl") 
104       
105      To save the in-memory Thesaurus to a Whoosh index... 
106       
107      >>> from whoosh.filedb.filestore import FileStorage 
108      >>> fs = FileStorage("index") 
109      >>> t.to_storage(fs) 
110       
111      To load a Thesaurus object from a Whoosh index... 
112       
113      >>> t = Thesaurus.from_storage(fs) 
114       
115      The Thesaurus object is thus usable in two ways: 
116       
117      * Parse the wn_s.pl file into memory (Thesaurus.from_*) and then look up 
118        synonyms in memory. This has a startup cost for parsing the file, and uses 
119        quite a bit of memory to store two large dictionaries, however synonym 
120        look-ups are very fast. 
121         
122      * Parse the wn_s.pl file into memory (Thesaurus.from_filename) then save it to 
123        an index (to_storage). From then on, open the thesaurus from the saved 
124        index (Thesaurus.from_storage). This has a large cost for storing the index, 
125        but after that it is faster to open the Thesaurus (than re-parsing the file) 
126        but slightly slower to look up synonyms. 
127       
128      Here are timings for various tasks on my (fast) Windows machine, which might 
129      give an idea of relative costs for in-memory vs. on-disk. 
130       
131      ================================================ ================ 
132      Task                                             Approx. time (s) 
133      ================================================ ================ 
134      Parsing the wn_s.pl file                         1.045 
135      Saving to an on-disk index                       13.084 
136      Loading from an on-disk index                    0.082 
137      Look up synonyms for "light" (in memory)         0.0011 
138      Look up synonyms for "light" (loaded from disk)  0.0028 
139      ================================================ ================ 
140       
141      Basically, if you can afford spending the memory necessary to parse the 
142      Thesaurus and then cache it, it's faster. Otherwise, use an on-disk index. 
143      """
